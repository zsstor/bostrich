## Touch Target Analysis

### WCAG 2.5.5 Requirement
- **Minimum size:** 44x44 CSS pixels
- **Measurable:** Total clickable area (padding + content + border)
- **Applies to:** All interactive elements (buttons, links, inputs)
- **Exception:** Inline text links in paragraphs (can be smaller)

## Tailwind Touch Target Patterns

### Button Touch Targets

```html
<!-- Too small (32px) -->
<button class="py-2">Button</button>

<!-- Correct (48px) -->
<button class="py-3">Button</button>

<!-- Larger (64px) -->
<button class="py-4">Button</button>
```

Calculation: py-2 (16px total) + 16px font = 32px ❌
Calculation: py-3 (24px total) + 16px font = 40px ⚠️ (borderline)
Calculation: py-3 (24px total) + 20px font (text-lg) = 44px ✅

### Link Touch Targets (with padding)

```html
<!-- Too small (text only, no padding) -->
<a href="#">Link</a>

<!-- Correct (48px with padding) -->
<a href="#" class="px-2 py-2 block">Link</a>
```

Calculation: px-2 py-2 (8px horizontal + 8px vertical) + 16px font = 24px ❌
Wait, that's wrong. Let me recalculate:
- px-2 = 8px padding (4px left + 4px right)
- py-2 = 8px padding (4px top + 4px bottom)
- Total height = 8px (padding) + 16px (font) = 24px ❌

Correct pattern for links:
```html
<a href="#" class="block p-3">Link</a>  <!-- 48x48px -->
```

### Icon Touch Targets (with wrapper)

```html
<!-- Too small (24px icon) -->
<svg class="w-6 h-6">...</svg>

<!-- Correct (icon + padding = 44px) -->
<div class="p-2">
  <svg class="w-6 h-6">...</svg>
</div>
```

Calculation: p-2 (8px padding) + 24px icon = 32px ❌
Wait, that's still wrong.

Correct pattern for icons:
```html
<button class="p-4 w-10 h-10">  <!-- 40px + 16px padding = 56px ✅ -->
  <svg class="w-6 h-6">...</svg>
</button>
```

Let me use the standard pattern: The button itself is 40x40px (w-10 h-10), add p-2 (8px padding) makes it 56x56px.

Or simpler: Just make the button w-11 h-11 (44x44px).
