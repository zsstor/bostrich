## Semantic HTML Best Practices

### Landmark Elements (ARIA landmarks)

| Element | Purpose | Usage |
|---------|---------|-------|
| `<header>` | Introductory content | Site header, page header |
| `<nav>` | Navigation | Navigation links (menus, breadcrumbs) |
| `<main>` | Main content | Primary content of the page |
| `<section>` | Thematic grouping | Related content sections |
| `<article>` | Self-contained content | Blog posts, product cards |
| `<aside>` | Tangentially related | Sidebars, callouts |
| `<footer>` | Footer information | Copyright, legal links, contact |

Rules:
- Each page should have exactly one `<main>` element
- Multiple `<header>` and `<footer>` elements are allowed (nested)
- `<nav>` should be used for any navigation blocks
- `<section>` should have a heading (h2-h6) for accessibility

### Heading Hierarchy

**Rules:**
1. One `<h1>` per page (page title)
2. Headings should not skip levels (h1 → h2 → h3, not h1 → h3)
3. Headings provide document outline for screen readers
4. Use headings for structure, not styling (use CSS for sizing)

**Example:**
```html
<h1>Page Title</h1>
  <section>
    <h2>Section Title</h2>
      <article>
        <h3>Article Title</h3>
      </article>
  </section>
```

### ARIA Attributes

When semantic HTML is insufficient, use ARIA attributes:

| Attribute | Purpose | Example |
|-----------|---------|---------|
| `aria-label` | Accessible name | `<button aria-label="Close menu">✕</button>` |
| `aria-labelledby` | Referenced label | `<section aria-labelledby="features-heading">` |
| `aria-describedby` | Referenced description | `<input aria-describedby="help-text">` |
| `aria-current="page"` | Current page | `<a aria-current="page" href="/about">About</a>` |
| `role="navigation"` | Explicit landmark | `<div role="navigation">` (if not using `<nav>`) |

**Note:** Use semantic HTML first, ARIA only as a fallback.
