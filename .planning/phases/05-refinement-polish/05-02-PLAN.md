## Alt Text Best Practices

### Descriptive Alt Text (for content images)
- Describe the image content and purpose
- Be specific but concise (125 characters or less preferred)
- Include context (what is this image showing?)
- Example: "Ladder with 8 rungs for vertical scaling"

### Empty Alt Text (for decorative images)
- Use `alt=""` to tell screen readers to ignore the image
- Do not omit alt attribute (screen readers will read filename)
- Example: `alt=""` for background patterns or decorative icons

### Complex Images (charts, graphs, infographics)
- Use alt text for brief description
- Provide detailed description in text below image
- Or use aria-describedby to link to long description

### SVG Icons (Lucide icons)
- Lucide components are SVG, not `<img>` tags
- No alt attribute possible on SVGs
- Use `aria-label` or `<title>` element for accessibility
- Example: `<Twitter aria-label="Twitter" />`

## Lucide Icon Accessibility

Lucide icons are Astro components that render SVGs. They do not support `alt` attributes. Instead, use:

1. **aria-label:** Provides accessible name for screen readers
   ```astro
   <Twitter aria-label="Twitter (opens in new tab)" />
   ```

2. **title:** Provides tooltip and accessible name
   ```astro
   <Twitter>
     <title>Twitter</title>
   </Twitter>
   ```

3. **role="img":** Explicitly indicates the element is an image
   ```astro
   <Twitter role="img" aria-label="Twitter" />
   ```

Best practice: Use aria-label on the icon component itself.
