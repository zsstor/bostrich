## Image Optimization Approach

### Why fetchpriority="high" on Hero Images
- Hero images are the first element users see on product pages
- They are the largest contentful paint (LCP) candidates
- Browsers prioritize loading based on viewport position and fetchpriority
- Setting fetchpriority="high" tells the browser to load immediately, not defer

### Why Responsive srcset
- Mobile users don't need 800px wide images (wastes bandwidth)
- Desktop users benefit from higher-resolution images
- srcset allows browser to select optimal size for current viewport
- Improves Core Web Vitals (LCP, CLS) and user experience

### Unsplash Image Optimization
Unsplash supports query parameters for optimization:
- `?w={width}` — Resize to specific width
- `?q={quality}` — Set compression quality (1-100)
- `?format=webp` — Serve WebP format (smaller than JPEG)
- `?auto=format` — Let Unsplash choose best format (WebP with AVIF fallback)

Example srcset pattern:
```
https://images.unsplash.com/photo-xxx?w=400&q=80 400w,
https://images.unsplash.com/photo-xxx?w=800&q=80 800w,
https://images.unsplash.com/photo-xxx?w=1200&q=80 1200w
```

## Image Loading Best Practices

1. **Hero images (above fold):**
   - `fetchpriority="high"` — Load immediately
   - NO `loading="lazy"` — Never lazy load above-fold content
   - Explicit `width` and `height` — Prevent layout shift

2. **Non-hero images (below fold):**
   - `loading="lazy"` — Defer until in viewport
   - Explicit `width` and `height` — Prevent layout shift
   - Optional srcset — If image is large or important

3. **Background images (CSS):**
   - Not in HTML, so no fetchpriority or loading attributes
   - Use CSS `image-rendering: auto` and `background-size: cover`

  width="800"
  height="600"
  fetchpriority="high"
  class="w-full h-96 object-cover rounded-lg"
/>
