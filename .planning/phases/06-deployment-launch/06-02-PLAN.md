  <files></files>
  <action>
  Human verification of deployment and domain configuration. Follow the steps below to verify the site is live and accessible at bostri.ch.

  **Step 1: Verify Vercel Deployment**

  If the CLI deployment succeeded, visit the Vercel dashboard to confirm:
  - Go to https://vercel.com/dashboard
  - Find the "bostrich" project
  - Click the latest deployment — it should show "Ready" status
  - Visit the auto-generated URL (e.g., bostrich.vercel.app) — site should load correctly

  **Step 2: Configure Custom Domain**

  1. In the Vercel dashboard, go to Project → Settings → Domains
  2. Add domain: `bostri.ch`
  3. Also add: `www.bostri.ch` (redirect to bostri.ch)
  4. Vercel will show the DNS records you need to configure

  **Step 3: Configure DNS Records**

  At your domain registrar's DNS management panel, add:
  - Type: CNAME, Name: `www`, Value: `cname.vercel-dns.com`
  - Type: A, Name: `@`, Value: `76.76.21.21`

  **Step 4: Verify SSL**

  After DNS propagates (can take up to 48 hours, often minutes):
  - Visit https://bostri.ch — should load with valid SSL certificate
  - Visit http://bostri.ch — should redirect to https://bostri.ch
  - Vercel automatically provisions SSL certificates

  **Step 5: Verify Pages**

  Check that all these URLs load correctly:
  - https://bostri.ch
  - https://bostri.ch/about
  - https://bostri.ch/pricing
  - https://bostri.ch/contact
  - https://bostri.ch/products
  - https://bostri.ch/products/ladder
  - https://bostri.ch/robots.txt
  - https://bostri.ch/sitemap-0-index.xml
  </action>

<task type="auto">
  <name>Task 3: Run Lighthouse audit on production</name>
  <files></files>
  <read_first>
    - astro.config.mjs (confirm site URL for audit target)
  </read_first>
  <action>
1. Install Lighthouse CLI:
   ```
   npm install -g lighthouse
   ```
   Or use npx without installing:
   ```
   npx lighthouse https://bostri.ch --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless"
   ```

2. Run Lighthouse audit against the production URL:
   ```
   npx lighthouse https://bostri.ch --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless" --only-categories=performance,accessibility,best-practices
   ```

   If Lighthouse CLI is not available or fails, use the web-based alternative:
   - Visit https://pagespeed.web.dev/analysis/https-bostri-ch/some-id
   - Or Chrome DevTools → Lighthouse tab

3. Parse the results and check scores:
   - Performance: must be > 80 (static site should score 90+)
   - Accessibility: must be > 85 (WCAG AA compliance from Phase 5)
   - Best Practices: must be > 85 (HTTPS, no console errors)

4. If scores are below thresholds:
   - Review the Lighthouse report for specific issues
   - Common fixes for static sites: image optimization (already done in Phase 5), render-blocking resources, unused CSS
   - For this Astro static site, scores should naturally be high since:
     * No JavaScript frameworks loaded by default
     * Images already optimized with fetchpriority and srcset
     * Semantic HTML already applied
     * Font-display: swap already configured

5. Clean up the report file if generated:
   ```
   rm -f lighthouse-report.json
   ```

6. Record the Lighthouse scores in the task summary.
  </action>
